本文目的是引导新手快速使用git进行版本管理，暂不涉及版本回退、分支等内容。

简介

Git已经流行好几年了，不过由于前些年多数用的是svn，所以刚使用git时可能会有些不习惯。
简单说下两者较大的区别

  • Git是分布式版本控制，每个开发者本机都有版本管理，可脱离服务端使用；而SVN是集中式版本控制，完全的C/S架构，无法脱离服务端。Git在迁移仓库时，比SVN要容易太多
  • Git独有的分支，可以高效切换，合并不同开发线

虽然Git不需要服务端就可以使用，但在多人协作时，通常还是会部署一台服务器，用来管理开发者权限和交换文件，我们这里姑且称之为服务端，所有人的提交都推送到服务端，同样也是从该服务端拉取其他人的提交内容。注意，该服务器只是充当一个权限管理，文件交换的作用，并不是必须的。

常见错误

下面是一些新手常见的错误，很好解决。由于笔者书写本文时，不能一一复现错误问题，所以打算在后续使用过程中，遇到的时候再添加进来。

git pull合并失败

原因：通常出现在版本库初始化后，向仓库添加新的文件，但是该文件已经有人创建提交到远端仓库了。
解决：使用git pull origin master --allow-unrelated-histories。或者使用git clone克隆一份最新的版本，然后再加入自己的文件（建议此办法）。

git pull文件冲突

使用git status查看状态：

查看文件，有冲突内容标注：

原因：其他人和自己修改了相同文件，git不知道应该保留谁的内容，只能手动解决
解决：打开文件，找到冲突位置，手动处理后，重新提交

git commit 报错

原因：没有设置git用户名，且系统无法自动根据主机信息设置
解决：按照提示，使用git config 设置

git push报错

原因：本地版本落后于远端，在没有使用git pull的前提下，直接git push。
解决：先git pull，再git push
