hexo-快速搭建自己的博客

发表于 2019-12-02 | 评论数：
本文字数： 4.1k | 阅读时长 ≈ 4 分钟

提起个人博客，恐怕用的最多的是wordpress ,也反复纠结过使用什么样的博客系统是最合适的，最终我选择了hexo。如果你觉得wordpress太重，如果你只是想写写文章，如果你不想重复造轮子，那么请你一定要知道Hexo这个神器

Hexo简介

hexo，是目前为止我用过的，最轻量、最简洁的博客系统，没有后台后台管理，不需要懂PHP、不需要懂Python。只需要懂Markdown语法,剩下的就是几行命令的事儿。

目录结构如下

_config.yml

网站配置信息，网站标题、作者等。下面列举一下常用配置参数，后面可以自定义设置。

参数 说明
title 站点标题
subtitle 站点副标题
description 站点描述
keywords 站点关键词
url 网址
permalink 文章永久链接格式
skip_render 跳过指定文件的渲染
theme 当前主题
deploy 站点发布相关的配置

package.json

应用程序包信息，一般无需修改

scaffolds

模板文件目录，使用hexo命令创建新的文章时，将会根据该目录下的模板来创建

source

存放用户文章内容的目录。我们创建的文章资源，都在该目录下。在执行生成网页的命令后，除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析，并放到根目录下的public 目录中，而其他文件会被拷贝过去

themes

存放主题的目录。Hexo可以定制主题，目前有很多优秀的开源主题可供使用，Hexo会根据主题来生成静态页面。
